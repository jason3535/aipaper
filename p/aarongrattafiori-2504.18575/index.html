摘要 · Abstract

由AI驱动的自主UI智能体在自动化日常任务（如报税和支付账单）方面具有巨大潜力，但安全问题是其充分发挥潜力的主要挑战，尤其是智能体能够代表用户采取行动。现有的针对Web智能体的提示注入测试要么过于简化威胁（测试不现实的场景或赋予攻击者过多能力），要么仅关注单步孤立任务。为了更准确地衡量安全Web智能体的进展，我们引入了WASP——一个用于端到端评估Web智能体对抗提示注入攻击安全性的新公开基准。使用WASP进行评估表明，即使是最顶尖的AI模型（包括具有高级推理能力的模型）也会在非常现实的场景中被简单、低人工成本的注入所欺骗。我们的端到端评估揭示了一个此前未被观察到的见解：尽管攻击在高达86%的案例中部分成功，但即使是最先进的智能体也常常难以完全实现攻击者的目标——这凸显了当前“因无能而安全”的状态。

Autonomous UI agents powered by AI have tremendous potential to boost human productivity by automating routine tasks such as filing taxes and paying bills. However, a major challenge in unlocking their full potential is security, which is exacerbated by the agent's ability to take action on their user's behalf. Existing tests for prompt injections in web agents either over-simplify the threat by testing unrealistic scenarios or giving the attacker too much power, or look at single-step isolated tasks. To more accurately measure progress for secure web agents, we introduce WASP -- a new publicly available benchmark for end-to-end evaluation of Web Agent Security against Prompt injection attacks. Evaluating with WASP shows that even top-tier AI models, including those with advanced reasoning capabilities, can be deceived by simple, low-effort human-written injections in very realistic scenarios. Our end-to-end evaluation reveals a previously unobserved insight: while attacks partially succeed in up to 86% of the case, even state-of-the-art agents often struggle to fully complete the attacker goals -- highlighting the current state of security by incompetence.

核心贡献 · Key contributions

  • 提出WASP，首个针对提示注入攻击的Web智能体安全端到端评估公开基准。
    Introduces WASP, the first public benchmark for end-to-end evaluation of web agent security against prompt injection attacks.
  • 揭示顶级AI模型在真实场景中可被简单、低成本的注入欺骗。
    Reveals that top-tier AI models can be deceived by simple, low-effort human-written injections in realistic scenarios.
  • 显示攻击在高达86%的案例中部分成功，但智能体常无法完全达成攻击者目标。
    Shows that attacks partially succeed in up to 86% of cases, yet agents often fail to fully complete attacker goals.
  • 强调当前Web智能体中的“无能安全”现象。
    Highlights the phenomenon of 'security by incompetence' in current web agents.
  • 提供现实且具挑战性的基准，以衡量安全Web智能体开发的进展。
    Provides a realistic and challenging benchmark to measure progress in secure web agent development.

局限 · Limitations

  • 基准仅关注提示注入攻击，未涵盖其他安全威胁。
    Benchmark focuses only on prompt injection attacks, not other security threats.
  • 评估限于Web智能体，结论可能不适用于其他类型智能体。
    Evaluation limited to web agents; findings may not generalize to other agent types.
  • 攻击由人工编写，自动化或自适应攻击可能产生不同结果。
    Attacks are human-written; automated or adaptive attacks may yield different results.
  • 基准场景可能未覆盖所有实际部署环境。
    Benchmark scenarios may not cover all real-world deployment contexts.
  • 随着模型改进，“无能安全”现象可能不会持续。
    Security by incompetence may not persist as models improve.
