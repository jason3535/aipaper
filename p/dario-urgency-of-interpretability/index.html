摘要 · Abstract

机制可解释性的简要历史

* A Brief History of Mechanistic Interpretability

核心贡献 · Key contributions

  • 论证可解释性对AI安全至关重要，将其比作模型的MRI。
    Argues interpretability is crucial for safe AI, comparing it to an MRI for models.
  • 描述机械可解释性的进展：特征、稀疏自编码器、回路。
    Describes progress in mechanistic interpretability: features, sparse autoencoders, circuits.
  • 展示特征操控（如金门大桥克劳德）以控制模型行为。
    Demonstrates feature manipulation (e.g., Golden Gate Claude) to control model behavior.
  • 展示回路追踪模型推理，例如达拉斯所在州的首府。
    Shows circuits trace model reasoning, e.g., capital of state containing Dallas.
  • 提出可解释性作为对齐的独立测试集，与训练分离。
    Proposes interpretability as an independent test set for alignment, separate from training.
  • 呼吁通过公司、学术界和政府政策加速可解释性研究。
    Calls for accelerating interpretability research via companies, academia, and government policies.

局限 · Limitations

  • 可解释性方法目前扩展性差；仅在小模型中发现一小部分特征。
    Interpretability methods currently scale poorly; only a fraction of features found in small models.
  • 叠加使许多特征不可解释；稀疏自编码器仅部分解决此问题。
    Superposition makes many features uninterpretable; sparse autoencoders only partially address this.
  • 回路发现仍为手动；自动化发现数百万回路具有挑战性。
    Circuit discovery remains manual; automating it for millions of circuits is challenging.
  • 用于风险检测的实际应用仍处于实验阶段；理论与应用之间存在差距。
    Practical use for risk detection is still experimental; gap between theory and application.
  • 与AI进展赛跑：可解释性可能在强大AI到来前（2026-2027）尚未成熟。
    Race against AI progress: interpretability may not mature before powerful AI arrives (2026-2027).
